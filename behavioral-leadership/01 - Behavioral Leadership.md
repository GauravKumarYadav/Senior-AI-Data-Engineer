# 🎤 Behavioral & Leadership

> **Phase:** 8 | **Duration:** ~1 week | **Priority:** Medium
> **Related:** All topics — this applies to how you discuss everything

---

## Checklist

### STAR Method Stories (Prepare 8-10)
- [ ] Situation → Task → Action → Result
- [ ] Quantify results: "reduced pipeline runtime by 60%", "saved $200K/year"

### Technical Leadership Stories
- [ ] Led a technical design/architecture decision
- [ ] Chose between competing technologies (and justified it)
- [ ] Disagreed with team/manager and reached resolution
- [ ] Mentored a junior engineer — specific growth outcome
- [ ] Dealt with technical debt — prioritized and addressed it
- [ ] Handled a production incident / data pipeline failure
- [ ] Drove adoption of a new tool/process across the team

### Communication
- [ ] Explaining data pipelines to PMs/analysts (non-technical)
- [ ] Writing RFC/design docs — structure and approach
- [ ] Presenting technical proposals to leadership
- [ ] Saying "no" with data — prioritizing requests

### Project Management
- [ ] Estimating timelines for data engineering projects
- [ ] Breaking down large projects into milestones
- [ ] Handling scope creep and shifting requirements
- [ ] Managing stakeholder expectations

### Culture & Values
- [ ] Amazon Leadership Principles (if targeting Amazon)
- [ ] Google's "Googleyness" — collaboration, humility
- [ ] Meta: move fast, be bold, build social value
- [ ] Company-specific values — research before each interview

### Common Behavioral Questions
- [ ] Tell me about a time you had to learn something quickly
- [ ] Describe a project you're most proud of
- [ ] How do you handle conflicting priorities from multiple stakeholders?
- [ ] Tell me about a time you made a mistake and how you handled it
- [ ] How do you approach a project with ambiguous requirements?
- [ ] Describe how you've improved a process or system
- [ ] Tell me about a time you had to influence without authority

### Incident Management
- [ ] Pipeline failure stories: what happened, how you detected, how you fixed
- [ ] Postmortem process: blameless, root cause, action items
- [ ] Monitoring and alerting improvements after incidents
- [ ] Communication during incidents: status updates, stakeholder management
