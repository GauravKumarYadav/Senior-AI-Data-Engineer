Activation Functions

вҡЎ ReLU

max(0, x) вҖ” Default for hidden layers. Cheap, avoids vanishing gradients.

рҹ§  GELU

x В· ОҰ(x) вҖ” Used in Transformers. Smoother ReLU, better gradient flow.

рҹ“Ҡ Sigmoid

1/(1+eвҒ»ЛЈ) вҖ” Binary output, gates. Range: (0, 1).

рҹҺҜ Softmax

eЛЈвҒұ / ОЈeЛЈКІ вҖ” Multi-class output. Sums to 1.

Loss Functions

рҹ’Ў Interview Insight

"Why do we need activation functions?" Without them, stacking layers is pointless вҖ” a composition of linear transformations is still linear (WвӮӮ(WвӮҒx + bвӮҒ) + bвӮӮ = W'x + b'). Activation functions introduce non-linearity, allowing the network to learn complex decision boundaries. The universal approximation theorem says a single hidden layer with non-linear activations can approximate any continuous function.

Backpropagation & Optimizers

Backpropagation: How Networks Learn

Forward Pass (compute predictions): Input вҶ’ Layer 1 вҶ’ Layer 2 вҶ’ ... вҶ’ Output вҶ’ Loss Backward Pass (compute gradients via chain rule): вҲӮLoss/вҲӮw = вҲӮLoss/вҲӮoutput В· вҲӮoutput/вҲӮhidden В· вҲӮhidden/вҲӮw Update weights: w_new = w_old - learning_rate Г— вҲӮLoss/вҲӮw

Optimizers Compared

SGD + Momentum

Basic gradient descent. Fine control, better generalization for CNNs. Needs careful LR tuning.

Adam

Adaptive LR per parameter + momentum. Default choice for most architectures.

AdamW

Adam with decoupled weight decay. Gold standard for Transformers and LLM fine-tuning.

# SGD with momentum optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.9) # Adam: adaptive learning rates (most popular) optimizer = optim.Adam(model.parameters(), lr=1e-3) # AdamW: better weight decay for Transformers optimizer = optim.AdamW(model.parameters(), lr=5e-5, weight_decay=0.01)

Learning Rate Schedules

# Cosine annealing: smooth decay scheduler = CosineAnnealingLR(optimizer, T_max=num_epochs, eta_min=1e-6) # OneCycle: warmup вҶ’ peak вҶ’ decay (best for training from scratch) scheduler = OneCycleLR(optimizer, max_lr=1e-3, total_steps=total_steps)

Batch Norm vs Layer Norm

BatchNorm

  • Normalizes across the batch dimension
  • Used in CNNs
  • Different behavior train vs eval
  • Depends on batch statistics

LayerNorm

  • Normalizes across feature dimension
  • Used in Transformers
  • Same behavior train and eval
  • Independent of batch size
рҹ’Ў Interview Insight

"Why is Adam the default for Transformers but SGD for CNNs?" Adam adapts the learning rate per parameter, ideal for Transformers where different attention heads need different LRs. SGD with momentum requires careful tuning but often generalizes slightly better for CNNs, reaching flatter minima. AdamW (decoupled weight decay) is the gold standard for fine-tuning pre-trained Transformers.

Weight Initialization

Xavier/Glorot

Var(w) = 2/(fan_in + fan_out) вҖ” Best with Sigmoid, Tanh activations.

He/Kaiming

Var(w) = 2/fan_in вҖ” Best with ReLU, LeakyReLU activations.

рҹ’Ў Interview Insight

"Dropout is 0.1 in BERT but 0.5 in older networks. Why?" BERT is pre-trained on massive data, so it's less prone to overfitting вҖ” light dropout suffices. Older, smaller networks trained on less data need aggressive dropout. The right rate depends on model size, data size, and task complexity.

CNN Basics вҖ” Convolutions, Pooling & ResNet

How Convolutions Work

Input Image (5Г—5) вҳ… Filter (3Г—3) = Feature Map (3Г—3) Slide the filter across the image: Element-wise multiply, then sum вҶ’ one output value Key params: kernel_size, stride, padding, in_channels, out_channels

Why CNNs? Parameter sharing (same filter everywhere), translation invariance (detects patterns regardless of position), and spatial hierarchy (edges вҶ’ textures вҶ’ objects).

Pooling

Max Pooling takes the maximum from each region, reducing spatial dimensions by 2Г—. Keeps the strongest activation.

"Why are skip connections so important?" They solve the degradation problem: gradients flow directly through the shortcut, enabling training of 100+ layer networks. Worst case, extra layers learn identity. This idea directly inspired residual connections in Transformers.

Self-Attention: The Core Mechanism

Step 1: Create Q, K, V from input embeddings Q = X В· W_Q (What am I looking for?) K = X В· W_K (What do I contain?) V = X В· W_V (What information do I provide?) Step 2: Attention scores = Q В· K^T / вҲҡd_k (dot product similarity) Step 3: Weights = softmax(Scores) Step 4: Output = Weights В· V Formula: Attention(Q, K, V) = softmax(QK^T / вҲҡd_k) В· V

Why scale by вҲҡd_k? Without scaling, dot products grow large for high dimensions, pushing softmax into near-zero gradient regions. Dividing keeps variance at ~1.

Multi-Head Attention

Head 1: Attention(QВ·WвӮҒбө , KВ·WвӮҒбөҸ, VВ·WвӮҒбөӣ) вҶ’ "Focus on syntax" Head 2: Attention(QВ·WвӮӮбө , KВ·WвӮӮбөҸ, VВ·WвӮӮбөӣ) вҶ’ "Focus on semantics" Head 3: Attention(QВ·WвӮғбө , KВ·WвӮғбөҸ, VВ·WвӮғбөӣ) вҶ’ "Focus on position" Concatenate all heads вҶ’ Linear projection вҶ’ Output d_model=768, h=12 heads вҶ’ d_k = 768/12 = 64 per head

Positional Encoding

рҹ“җ Sinusoidal (Original)

Fixed, deterministic. Generalizes to longer sequences.

рҹ“ҡ Learned (BERT)

Trainable embeddings. Limited to max_position_embeddings.

рҹ”„ RoPE (Llama)

Rotary position вҖ” encodes relative position. Supports extrapolation.

рҹ’Ў Interview Insight

"Walk through a single Transformer encoder layer." Input + positional encoding вҶ’ (1) Multi-head self-attention вҶ’ Add & LayerNorm (residual) вҶ’ (2) Feed-forward network (two linear layers + GELU) вҶ’ Add & LayerNorm вҶ’ Output. Pattern: output = LayerNorm(x + Sublayer(x)).

Pre-Training & Transfer Learning

Pre-Training Objectives

Masked Language Model (BERT): Input: "The [MASK] sat on the [MASK]" Target: "The cat sat on the mat" вҶ’ Bidirectional: sees context on BOTH sides Causal Language Model (GPT): Input: "The cat sat" Target: "cat sat on" вҶ’ Autoregressive: only sees LEFT context Span Corruption (T5): Input: "The <X> on the mat" Target: "<X> cat sat <Y>" вҶ’ Encoder-decoder: flexible input/output

Transfer Learning Strategies

рҹ§Ҡ Feature Extraction

  • Pre-trained model: FROZEN
  • Only new head trains
  • Use when: tiny dataset, similar task

рҹ”Ҙ Fine-Tuning

  • Pre-trained model: UNFROZEN (small LR)
  • Everything trains
  • Use when: moderate data, best performance
# Differential learning rates for fine-tuning optimizer = optim.AdamW([ {'params': model.bert.parameters(), 'lr': 2e-5}, # Pre-trained: tiny LR {'params': model.classifier.parameters(), 'lr': 1e-3}, # New head: normal LR ])
рҹ’Ў Interview Insight

"When would you fine-tune vs use zero-shot/few-shot?" Fine-tune when you have labeled data (hundreds+), need high accuracy, task is specialized. Zero/few-shot when no labeled data, rapid prototyping. Fine-tuning a small model often beats few-shot with a large model and is cheaper at inference. The trend is toward efficient fine-tuning (LoRA, QLoRA).

рҹ’Ў Interview Insight

"Why optimizer.zero_grad() before loss.backward()?" PyTorch accumulates gradients by default (useful for gradient accumulation with large effective batch sizes). If you don't zero them, gradients from the previous batch add to the current ones. The pattern: zero_grad вҶ’ forward вҶ’ loss вҶ’ backward вҶ’ step.

Mixed Precision & Distributed Training

Mixed Precision Training

Uses float16 for forward/backward (faster, less memory) and float32 for weight updates (numerical stability). ~2Г— faster, ~50% less memory.

from torch.cuda.amp import autocast, GradScaler scaler = GradScaler() for batch_X, batch_y in dataloader: optimizer.zero_grad() with autocast(): logits = model(batch_X) loss = criterion(logits, batch_y) scaler.scale(loss).backward() scaler.unscale_(optimizer) torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0) scaler.step(optimizer) scaler.update()

Practical Troubleshooting

рҹ“ү Loss not decreasing

Lower LR, check data loading, verify labels.

рҹ’Ҙ NaN loss

Check for log(0), div by 0, reduce LR.

рҹ“Ҳ Overfitting

Dropout, weight decay, more data, early stop.

рҹ’ҫ OOM

Smaller batch, gradient accumulation, mixed precision.

рҹ’Ў Interview Insight

"How would you train a model that doesn't fit in GPU memory?" (1) Gradient accumulation: simulate large batch over N small batches. (2) Mixed precision: cut memory ~50%. (3) Gradient checkpointing: recompute activations during backward pass. (4) Model parallelism: split model across GPUs. (5) DeepSpeed/FSDP: automated model sharding. Start with the simplest solution first.
