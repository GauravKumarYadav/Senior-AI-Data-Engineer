🤖 AI & ML Foundations

🔥 ML Design Questions

These test your ability to think end-to-end: from problem framing to deployment. No hand-holding.

Q1: "Design a model to detect fraudulent credit card transactions."

  • Framing: Binary classification, extreme imbalance (~0.1-0.5% fraud), real-time <100ms
  • Features: Amount, velocity (txns in last 1hr/24hr), deviation from avg, geo distance, behavioral
  • Model: XGBoost/LightGBM (NOT deep learning for tabular). Need interpretability + fast inference
  • Imbalance: SMOTE or class_weight='balanced'. Optimize AUC-PR, not accuracy
  • Threshold: cost_FN (missed fraud) >> cost_FP (blocked legit) → low threshold
  • Deploy: Feature store + model server, A/B test, monitor concept drift, human-in-the-loop

Q2: "Build a recommendation system for e-commerce."

  • Collaborative Filtering: Finds surprising items, but cold start problem
  • Content-Based: No cold start for items, but filter bubble
  • Two-Tower (modern): User encoder → emb, Item encoder → emb, Score = dot product
  • Two-stage: Candidate generation (ANN on embeddings → ~1000 items) → Ranking (LightGBM/neural ranker → top 20)

Q3: "Design an ML system to predict customer churn."

  • Target definition matters: No activity in 30 days? 90 days? Business decision!
  • Feature windows: Lookback (last 90d) to predict churn in prediction window (next 30d). Never leak future data!
  • Action: Rank by P(churn) × customer_value. Target high-value, high-risk with retention offers

Q4: "Design an email spam classifier."

  • Features: TF-IDF on body, sender reputation, header/URL analysis, HTML/text ratio
  • Model: Start simple (Logistic Regression + TF-IDF), then gradient boosted trees
  • Precision > Recall: False positives (legit in spam) worse than false negatives
  • Adversarial: Spammers adapt! Need regular retraining, drift monitoring

Q5: "500 features, 10K rows. Walk through your approach."

  • EDA: Class distribution, missing values, correlation matrix → drop >0.95 correlated
  • Feature Selection (500→~50-100): Mutual information, Lasso (L1), RFE with XGBoost
  • Model: Baseline: Logistic Regression. Primary: XGBoost with strong regularization
  • Validation: 5-fold stratified CV (10K too few for hold-out). Pipeline to prevent leakage
  • Interpretability: SHAP values + permutation importance

Q6–Q8: Content Moderation / Search Ranking / Demand Forecasting

  • Content Moderation: Multi-label classification, fine-tuned RoBERTa, human-in-the-loop for edge cases, distilled model for latency
  • Search Ranking: Learning-to-Rank (LTR), BM25 + semantic → re-rank. Metrics: NDCG@10, MRR, MAP
  • Demand Forecasting: LightGBM with temporal features beats ARIMA. Hierarchy: SKU×Store×Day → reconcile up. Metric: WMAPE

📊 Evaluation & Metrics Traps

Q1: "Your model has 95% accuracy on binary classification. Is it good?"

A) Yes — 95% accuracy is excellent for any task
B) It depends on class distribution — with 95/5 imbalance, a dummy model predicting majority class also gets 95%
C) No — you always need at least 99% accuracy

Q2: "Your AUC-ROC is 0.99 but AUC-PR is 0.15. What's happening?"

A) The model is overfitting to the training data
B) Extreme class imbalance — massive TN count inflates AUC-ROC. AUC-PR reveals terrible precision relative to the small positive class
C) The metrics are calculated incorrectly — they should be similar

Precision vs Recall — When to Optimize Which

🎯 Optimize Precision

  • Spam filtering (blocking legit email = bad UX)
  • Content recommendations (irrelevant = frustration)
  • Drug approval (false approval = patient risk)

🔍 Optimize Recall

  • Cancer screening (missing cancer = harm)
  • Fraud detection (missing fraud = loss)
  • Security threats (missing = breach)
🎯 Setting Classification Thresholds

Default 0.5 is almost never optimal. Better: Cost-based (threshold = C_FP / (C_FP + C_FN)), Precision-at-recall ("I need 95% recall"), F1-optimal, or Business constraint ("max 1% FPR").

"Model performs great on validation but poorly in production?"

  • 1. Data drift: Production distribution differs from training
  • 2. Data leakage: Accidentally used future data or target info
  • 3. Preprocessing mismatch: Different scaling/encoding in serving
  • 4. Concept drift: Feature-target relationship changed over time
  • Debug: Compare distributions, check leakage, verify pipeline, shadow mode

Type I vs Type II Errors (Medical Test)

  • Type I (False Positive): Test says POSITIVE, reality HEALTHY → unnecessary treatment, anxiety
  • Type II (False Negative): Test says NEGATIVE, reality SICK → missed diagnosis, progression
  • Which is worse? Depends! Cancer: Type II worse. Spam: Type I worse.

🧠 Deep Learning & NLP Questions

Q1: "Explain the attention mechanism. Why better than RNNs?"

  • RNN Problem 1: Sequential processing → can't parallelize. Attention: all tokens simultaneously
  • RNN Problem 2: Vanishing gradients → long-range deps lost. Attention: O(1) path length
  • RNN Problem 3: Information bottleneck in fixed hidden state. Attention: every token ↔ every token
  • Cost: O(n²) memory/compute → why context windows are limited. Solutions: Flash Attention, sparse attention

Self-Attention

  • Q, K, V from SAME input
  • Each word attends to every other
  • Used in: BERT encoder, GPT decoder

Cross-Attention

  • Q from one source, K/V from another
  • Decoder attends to encoder output
  • Used in: T5/BART, image captioning

Q3: "Why positional encoding in Transformers?"

  • Self-attention is permutation-equivariant — treats input as a set, not a sequence
  • Without it: "dog bites man" = "man bites dog" (identical representations!)
  • Sinusoidal: Fixed, generalizes to longer sequences
  • Learned (BERT): Trainable, limited to max_position
  • RoPE (Llama): Relative position via rotation, supports extrapolation

Q4: "How is BERT pre-trained?"

  • MLM: Mask 15% of tokens (80% [MASK], 10% random, 10% unchanged). Predict originals → bidirectional understanding
  • NSP: Are two sentences consecutive? 50% real, 50% random pairs. (RoBERTa later dropped this — not very useful)
  • Data: BooksCorpus + Wikipedia (~16GB). Cost: 4 days on 16 TPUs

Q5: "Vanishing gradient problem — how do modern architectures solve it?"

  • Gradients shrink exponentially through many layers → product of terms <1 → 0
  • Solutions: ReLU (gradient=1 for positive), residual connections (gradient highway), layer norm, careful init (He/Kaiming), gradient clipping

🔥 Catastrophic Forgetting

  • Small LR (2e-5): gentle updates
  • Warmup: don't shock weights
  • Gradual unfreezing: head first
  • LoRA: only train small adapters

⚡ Full Fine-Tune vs LoRA

  • Full: update ALL params (e.g., 7B)
  • LoRA: ΔW = A×B, rank r≪d
  • ~0.1% trainable parameters
  • QLoRA: + 4-bit quantization

⚡ Rapid Fire — 20 Q&A

Click each question to reveal the answer. 2-3 sentences max — testing breadth and precision.

1. What is the bias-variance tradeoff?
Bias = underfitting (too simple); variance = overfitting (too complex). Total error = bias² + variance + irreducible noise. Find the complexity sweet spot.
2. Why is cross-validation better than a single split?
Uses all data for both training and validation across K folds. Mean ± std tells you both performance and stability. Critical for small datasets.
3. What does L1 do that L2 doesn't?
L1 drives weights to exactly zero → automatic feature selection. L2 only shrinks toward zero. Use L1 when many features are irrelevant.
4. XGBoost vs Random Forest?
RF = bagging (parallel trees, majority vote) reduces variance. XGBoost = boosting (sequential trees, correct previous errors) reduces bias. XGBoost also has built-in regularization + handles missing values.
5. What is data leakage?
Information from outside training (test data, future data, target variable) leaks in. Causes: fitting scalers on full data, target-derived features, ignoring time order. Pipelines prevent most leakage.
6. What activation do Transformers use?
GELU in feed-forward layers — smooth approximation of ReLU allowing small negative values for better gradient flow. Newer models use SwiGLU.
7. Purpose of the attention mask?
Tells the model which tokens to attend to. Padding tokens get mask=0 so they don't influence output. In GPT, also prevents attending to future tokens.
8. Why AdamW over Adam?
AdamW decouples weight decay from gradients, applying it directly to weights. Standard Adam's decay interacts with adaptive LR in unintended ways. Better generalization.
9. model.train() vs model.eval()?
train() enables dropout + uses batch stats for BatchNorm. eval() disables dropout + uses stored running stats. Always switch to eval before inference.
10. Explain BPE in one sentence.
Start with individual characters and iteratively merge the most frequent adjacent pair into a new token until reaching the target vocabulary size.
11. BERT vs GPT?
BERT = encoder, bidirectional, great for understanding/classification. GPT = decoder, autoregressive, great for generation. BERT uses MLM; GPT uses next token prediction.
12. What is learning rate warmup?
Gradually increasing LR from near-zero to target over initial steps. Prevents large, destructive weight updates when gradients are noisy at the start.
13. When AUC-PR over AUC-ROC?
When classes are imbalanced. AUC-ROC can be misleadingly high due to massive true negatives. AUC-PR focuses on the positive class for an honest picture.
14. What is gradient clipping?
Limiting gradient magnitude during backprop (max_norm=1.0). Prevents exploding gradients that cause instability. Doesn't change direction, only scales if too large.
15. Cold start problem?
New users/items have no interaction history → can't personalize. Solutions: content-based features, popularity fallback, onboarding preferences.
16. What is SHAP?
SHapley Additive exPlanations — theoretically grounded feature attributions for individual predictions. Shows how much each feature pushed prediction up/down. Essential for interpretability.
17. Mixed precision training?
float16 for forward/backward (faster, less memory), float32 for weight updates (stability). ~2x speedup, ~50% memory reduction. Use torch.cuda.amp.
18. Bi-encoder vs cross-encoder?
Bi-encoder: encode texts independently (fast, scalable, pre-computable). Cross-encoder: process both jointly (more accurate, O(n²)). Use bi for retrieval, cross for re-ranking.
19. How does dropout work at inference?
It doesn't — disabled during inference. All neurons active but scaled by (1 - dropout_rate) to maintain expected magnitude. model.eval() handles this.
20. What is LoRA and why does it matter?
Freezes original weights, trains small rank-decomposed matrices (ΔW = A×B, r≪d). ~99% fewer trainable params. Enables fine-tuning large models on consumer GPUs. QLoRA adds 4-bit quantization.

🏆 Interview Cheat Sheet

1. Problem First

"What are we optimizing? What's the business metric?"

2. Ask About Data

"How much labeled data? Class balance? Feature types?"

3. Never Say "Accuracy"

For imbalanced problems → AUC-PR, F1, recall. Show you know WHY.

4. Prevent Leakage

Pipelines, proper splits, temporal awareness. Always mention this.

5. Bias-Variance

Every regularization technique maps to this concept. Know it cold.

6. Trees for Tabular

XGBoost/LightGBM + Optuna is the winning formula.

7. Transformers for Text

BERT for understanding, GPT for generation.

8. Evaluate First

Metric selection drives every decision downstream.

9. Deployment Matters

Latency, throughput, monitoring, drift detection.

10. Be Honest

"This approach is simpler but sacrifices X." Tradeoffs!
