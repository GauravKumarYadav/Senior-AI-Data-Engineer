🧠 What Is Dynamic Programming?

"Those who cannot remember the past are condemned to repeat it." — George Santayana (and every recursive function without memoization)

DP is an optimization technique that solves complex problems by breaking them into simpler, overlapping subproblems. Two properties must hold:

🔄 Overlapping Subproblems

The same smaller problem is solved many times during recursion. Computing fib(5) naïvely: fib(3) is computed twice, fib(2) three times. DP stores these results so each subproblem is solved exactly once.

🏗️ Optimal Substructure

An optimal solution can be constructed from optimal solutions to its subproblems. If the shortest path A→C goes through B, then A→B must itself be the shortest path from A→B.

The Recognition Test

🧩 Identical Subproblems?

Can you break it into smaller versions of the same problem? → DP candidate.

📊 Min/Max/Count?

Does the problem ask for minimum, maximum, or count of something? → Strong DP signal.

🤔 Choices at Each Step?

Take/skip, left/right, use/don't use? → DP with decision at each state.

💥 Exponential → Overlap?

Brute-force is exponential but recursion tree has repeated nodes? → DP will optimize.

🔀 Order Doesn't Matter?

Only the end state matters, not how you got there? → DP over greedy.

💡 Interview Cue

When an interviewer says "Can you optimize your brute-force solution?" and your recursion tree has repeated nodes — that's your cue to reach for DP.

🔄 Memoization vs Tabulation

Every DP problem can be solved two ways. Master both — interviewers may ask you to convert. The underlying recurrence is identical.

⬇️ Top-Down (Memoization)

  • 🔄 Write natural recursive solution
  • 💾 Add cache for solved subproblems
  • ✅ Intuitive, mirrors recurrence
  • ✅ Only solves needed subproblems
  • ⚠️ Recursion stack overhead
  • ⚠️ Stack overflow for large n

⬆️ Bottom-Up (Tabulation)

  • 📊 Build table from base cases up
  • 🔄 Fill iteratively, no recursion
  • ✅ No stack overhead
  • ✅ Cache-friendly memory access
  • ✅ Easier to optimize space
  • ⚠️ May compute unneeded subproblems

Fibonacci — Top-Down

from functools import lru_cache @lru_cache(maxsize=None) def fib(n: int) -> int: if n <= 1: return n return fib(n - 1) + fib(n - 2)

Fibonacci — Bottom-Up

def fib(n: int) -> int: if n <= 1: return n dp = [0] * (n + 1) dp[1] = 1 for i in range(2, n + 1): dp[i] = dp[i - 1] + dp[i - 2] return dp[n]

Space Optimization — O(1)

def fib(n: int) -> int: if n <= 1: return n prev2, prev1 = 0, 1 for _ in range(2, n + 1): prev2, prev1 = prev1, prev2 + prev1 return prev1 # When dp[i] depends only on a fixed window → reduce to O(1)
💡 Always Mention Space Optimization

Present the full table first, then optimize. It shows depth of understanding and interviewers love it. The progression: brute-force → memoization → tabulation → space-optimized.

🧩 Climbing Stairs & House Robber

Problem 1: Climbing Stairs (LeetCode 70)

You climb a staircase with n steps, taking 1 or 2 steps at a time. How many distinct ways to reach the top? This is Fibonacci in disguise.

State: dp[i] = distinct ways to reach step i
Recurrence: dp[i] = dp[i-1] + dp[i-2]

def climb_stairs(n: int) -> int: """Distinct ways to climb n stairs.""" if n <= 2: return n prev2, prev1 = 1, 2 for _ in range(3, n + 1): prev2, prev1 = prev1, prev2 + prev1 return prev1 # O(n) time, O(1) space
📌 dp[i] = ways to reach step i
 
🛑 Base: 1 step = 1 way, 2 steps = 2 ways
 
📊 dp[0]=1, dp[1]=2 (space-optimized)
🔄 Each step: sum of two previous
🏆 Answer is prev1
 
💡 For 1/2/3 steps: add dp[i-3]

Problem 2: House Robber (LeetCode 198)

Rob houses along a street — can't rob adjacent houses. Maximize total money. At each house: take (skip prev) or skip (keep prev best).

State: dp[i] = max money from first i houses
Recurrence: dp[i] = max(dp[i-1], dp[i-2] + nums[i])

def rob(nums: list[int]) -> int: """Max money without robbing adjacent.""" if not nums: return 0 if len(nums) == 1: return nums[0] prev2, prev1 = nums[0], max(nums[0], nums[1]) for i in range(2, len(nums)): prev2, prev1 = prev1, max( prev1, # skip house i prev2 + nums[i] # rob house i ) return prev1 # O(n) time, O(1) space
📌 dp[i] = max money from first i houses
 
🛑 Edge cases
 
 
📊 Base: dp[0]=nums[0], dp[1]=max(first two)
🔄 For each house:
🤔 Two choices:
⏭️ Skip → keep previous best
💰 Rob → prev2 + this house's money
 
🏆 Take the maximum
💡 House Robber II (circular): run twice
🎯 The Take/Skip Pattern

House Robber is the archetype of take/skip DP: at each element, you either include it (with a constraint) or exclude it. This pattern appears in dozens of problems: partition equal subset sum, target sum, and more. Master this one and you've unlocked a whole family.

🧩 Coin Change & Unique Paths

Problem 3: Coin Change (LeetCode 322)

Fewest coins to make amount. Each denomination can be used unlimited times. Classic unbounded knapsack variant.

State: dp[a] = min coins for amount a
Recurrence: dp[a] = min(dp[a - coin] + 1) for each coin

def coin_change(coins, amount): """Fewest coins to make amount, or -1.""" dp = [float("inf")] * (amount + 1) dp[0] = 0 for a in range(1, amount + 1): for coin in coins: if coin <= a: dp[a] = min(dp[a], dp[a - coin] + 1) return dp[amount] if dp[amount] != float("inf") else -1 # O(amount × coins) time, O(amount) space
📌 dp[a] = min coins for amount a
 
♾️ Init all to infinity (not yet achievable)
🛑 Base: 0 coins for amount 0
 
🔄 For each amount from 1 to target:
🪙 Try every coin denomination
✅ Coin fits? Check if it's better
📊 dp[a-coin]+1 = use this coin + best remainder
 
 
🎯 Still infinity? Impossible → -1
💡 Coin Change II asks for # of combos

Problem 4: Unique Paths (LeetCode 62)

Robot on m × n grid, can only move right or down. Count unique paths to bottom-right.

State: dp[r][c] = paths from (0,0) to (r,c)
Recurrence: dp[r][c] = dp[r-1][c] + dp[r][c-1]

Grid visualization (3×4): Start → 1 1 1 1 1 2 3 4 1 3 6 10 ← Answer! Each cell = sum of cell above + cell to the left
def unique_paths(m: int, n: int) -> int: """Unique paths in m×n grid. Space-optimized.""" row = [1] * n # top edge: all 1s for r in range(1, m): for c in range(1, n): row[c] += row[c - 1] return row[-1] # O(m×n) time, O(n) space
📌 dp[r][c] = paths to (r, c)
 
📏 Top edge: only one way (keep going right)
 
🔄 For each remaining row:
➕ row[c] += row[c-1] (above + left)
🧠 row[c] still holds "above" value!
 
🏆 Bottom-right = answer
💡 Obstacles? Set dp[r][c]=0 for them
💡 Grid DP Skeleton

Grid DP problems share the exact same skeleton — only the combination function changes: sum (Unique Paths), min (Min Path Sum), max (Max Gold). Recognize the skeleton and swap the operator.

🧩 Edit Distance — Full Walkthrough

LeetCode 72: Minimum operations (insert, delete, replace) to convert word1 into word2. The quintessential 2-string DP problem.

State: dp[i][j] = min edits to convert word1[0..i-1] → word2[0..j-1]

Grid Walkthrough: "horse" → "ros"

"" r o s "" [ 0 1 2 3 ] h [ 1 1 2 3 ] Three operations at each cell: o [ 2 2 1 2 ] • Replace: dp[i-1][j-1] + 1 r [ 3 2 2 2 ] • Delete: dp[i-1][j] + 1 s [ 4 3 3 2 ] • Insert: dp[i][j-1] + 1 e [ 5 4 4 3 ] If chars match: dp[i-1][j-1] (free!) Answer: 3 operations (replace h→r, delete r, delete e)
def min_distance(word1, word2): m, n = len(word1), len(word2) dp = [[0]*(n+1) for _ in range(m+1)] for i in range(m+1): dp[i][0] = i for j in range(n+1): dp[0][j] = j for i in range(1, m+1): for j in range(1, n+1): if word1[i-1] == word2[j-1]: dp[i][j] = dp[i-1][j-1] else: dp[i][j] = 1 + min( dp[i-1][j-1], # replace dp[i-1][j], # delete dp[i][j-1], # insert ) return dp[m][n]
📌 dp[i][j] = edits for word1[:i] → word2[:j]
 
📊 Create (m+1) × (n+1) table
 
🛑 Base: deleting all chars
🛑 Base: inserting all chars
 
🔄 Fill table row by row
 
✅ Characters match → free! No edit needed
 
🤔 Characters differ → best of 3 options:
🔄 Replace this char
❌ Delete from word1
➕ Insert into word1
 
🏆 Answer at dp[m][n]

Space-Optimized Version

def min_distance_optimized(word1, word2): """Two-row edit distance — O(min(m, n)) space.""" m, n = len(word1), len(word2) if m < n: return min_distance_optimized(word2, word1) prev = list(range(n + 1)) curr = [0] * (n + 1) for i in range(1, m + 1): curr[0] = i for j in range(1, n + 1): if word1[i-1] == word2[j-1]: curr[j] = prev[j-1] else: curr[j] = 1 + min(prev[j-1], prev[j], curr[j-1]) prev, curr = curr, prev return prev[n] # Time: O(m×n) | Space: O(min(m, n))
🎯 Interview Strategy

Walk through the grid on the whiteboard with a small example. Explain what each operation means physically: "Insert means we still need to match word1[0..i-1] against a shorter prefix of word2." Interviewers want to see you understand the recurrence, not just memorize code.

🔗 LCS & Knapsack Pattern

Longest Common Subsequence (LeetCode 1143)

State: dp[i][j] = LCS length of text1[0..i-1] and text2[0..j-1]

"abcde" vs "ace" → LCS = "ace" (length 3) "" a c e "" [ 0 0 0 0 ] a [ 0 1 1 1 ] Match → dp[i-1][j-1] + 1 b [ 0 1 1 1 ] No match → max(dp[i-1][j], dp[i][j-1]) c [ 0 1 2 2 ] d [ 0 1 2 2 ] e [ 0 1 2 3 ] ← Answer!
def longest_common_subsequence(text1, text2): m, n = len(text1), len(text2) dp = [[0] * (n + 1) for _ in range(m + 1)] for i in range(1, m + 1): for j in range(1, n + 1): if text1[i-1] == text2[j-1]: dp[i][j] = dp[i-1][j-1] + 1 else: dp[i][j] = max(dp[i-1][j], dp[i][j-1]) return dp[m][n] # O(m×n) time, O(m×n) space — can optimize to O(min(m,n))

Problems That Reduce to Knapsack

🎒 Subset Sum

Values = weights, target = sum. Can you pick items to reach exactly target?

⚖️ Partition Equal Subset

Target = total_sum / 2. Reduces to 0/1 knapsack.

🪙 Coin Change (Bounded)

Each coin used at most once → 0/1 knapsack. Unlimited → unbounded.

✂️ Rod Cutting

Unbounded knapsack. Maximize revenue from cutting a rod.

Space-Optimized 0/1 Knapsack

def knapsack_1d(weights, values, capacity): """Space-optimized 0/1 knapsack.""" dp = [0] * (capacity + 1) for i in range(len(weights)): for w in range(capacity, weights[i] - 1, -1): # Backwards! dp[w] = max(dp[w], dp[w - weights[i]] + values[i]) return dp[capacity]
💡 Why Loop Backwards?

If we iterate forward, dp[w - weights[i]] may already reflect item i (not the previous row), effectively picking item i multiple times — that's the unbounded knapsack. Backwards iteration preserves the "previous row" values. Explaining this shows genuine understanding.

🧠 Palindromic Subsequence Trick

Longest Palindromic Subsequence = LCS of the string and its reverse. No new DP needed: longest_common_subsequence(s, s[::-1]). This is why mastering core patterns pays off — reductions are free.

📋 DP Pattern Recognition Cheat Sheet

📏 Linear (1-D)

Clues: "steps," "houses," single sequence
State: dp[i]
Examples: Climbing Stairs, House Robber

🎯 Target Sum

Clues: "make amount," "reach target"
State: dp[amount]
Examples: Coin Change, Perfect Squares

📊 Grid (2-D)

Clues: "grid," "matrix," "right/down"
State: dp[r][c]
Examples: Unique Paths, Min Path Sum

🔤 Two-String

Clues: Two input strings, compare/transform
State: dp[i][j]
Examples: Edit Distance, LCS

🎒 0/1 Knapsack

Clues: "pick or skip," capacity constraint
State: dp[i][w]
Examples: Subset Sum, Partition

🔄 Unbounded Knapsack

Clues: "unlimited supply," reuse allowed
State: dp[w]
Examples: Coin Change, Rod Cutting

🎯 The 5-Step DP Framework

Write these on the whiteboard before coding. It shows structured thinking and buys time to think without awkward silence.

1️⃣ Define the State

What does dp[i] (or dp[i][j]) represent? This is the hardest step.

2️⃣ Find the Recurrence

How does dp[i] relate to smaller subproblems?

3️⃣ Base Cases

What are the trivial answers? dp[0], dp[0][0], etc.

4️⃣ Iteration Order

Which direction fills the table correctly?

5️⃣ Optimize Space

Can you go from 2-D → 1-D, or 1-D → O(1)?
