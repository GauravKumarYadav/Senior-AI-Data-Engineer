🎯 The Two-Pointer Technique

Master the foundational array and string techniques that appear in 40–50% of all coding interviews. The two-pointer technique is your first weapon — dead simple, insanely powerful.

The core idea: place one pointer at the start and one at the end. Based on the current result, decide which pointer to move inward. Because each pointer only moves in one direction, the total work is O(n).

How the Pointers Move

⬅️ Sum Too Small

Move left pointer right to increase the sum. We need a bigger number.

➡️ Sum Too Large

Move right pointer left to decrease the sum. We need a smaller number.

✅ Match Found

Record the result. Move either or both pointers depending on whether you need all pairs or just one.

Pointer Variations

↔️ Opposite Direction

Sorted two-sum, palindrome checking, container with most water, trapping rain water.

➡️➡️ Same Direction

Fast/slow pointer for linked lists, removing duplicates in-place, partitioning arrays.

🔱 Three Pointers

Dutch National Flag (sort colors), 3Sum (fix one + two-pointer on rest).

💡 Why Two Pointers Always Works

Suppose the answer is at indices (i, j). If left < i, then nums[left] + nums[right] ≤ target — we move left right. If right > j, we move right left. We never skip over the answer because we only move a pointer when the current sum tells us to. This proof intuition is interview gold.

🪟 The Sliding Window

Maintain a window of elements and slide it across the array, updating state incrementally instead of recomputing from scratch. Think of a caterpillar crawling — the front extends, the back catches up.

Fixed-Size Window

When the problem says "subarray of size k," the template is beautifully simple:

# Fixed sliding window template # 1. Compute initial window (first k elements) window_sum = sum(nums[:k]) max_sum = window_sum # 2. Slide: add right, remove left for i in range(k, len(nums)): window_sum += nums[i] - nums[i - k] max_sum = max(max_sum, window_sum) # Each slide costs O(1) → total O(n)

Variable-Size Window

When you need the shortest or longest subarray satisfying a condition:

left = 0 for right in range(len(arr)): # Expand: add arr[right] to window state while window_is_invalid(): # Shrink: remove arr[left] from window state left += 1 # Update answer here
🎯 Longest vs Shortest — Where to Update

Longest subarray: update the answer after the shrink loop (window is valid, maximize). Shortest subarray: update inside the shrink loop (window is valid, minimize before shrinking further). This trips up so many people!

⚠️ When Sliding Window Fails

If the array has negative numbers and you need a target sum, sliding window breaks — shrinking doesn't guarantee the sum increases. The monotonic property is lost. Use prefix sums + hashmap instead.

📊 Prefix Sums & Kadane's Algorithm

Prefix Sum — O(1) Range Queries

A prefix sum array P where P[i] = nums[0] + … + nums[i-1] lets you compute any range sum in O(1): sum(nums[i:j]) = P[j] - P[i]

def build_prefix_sum(nums: list[int]) -> list[int]: """P[0] = 0, P[i] = sum(nums[:i]).""" prefix = [0] * (len(nums) + 1) for i in range(len(nums)): prefix[i + 1] = prefix[i] + nums[i] return prefix

Prefix Sum + HashMap Pattern

For "subarray sum equals K" with negative numbers: maintain a running prefix_sum. At each index, check if prefix_sum - K exists in a hashmap of previously seen prefix sums. If it does — bingo, there's a subarray ending here with sum K.

💡 Why This Works

If prefix_sum[j] - prefix_sum[i] == K, then subarray nums[i+1..j] sums to K. By storing prefix sums in a hashmap as we go, we turn an O(n²) enumeration into O(n) lookups. This handles negatives gracefully — something sliding window cannot.

Kadane's Algorithm — Max Subarray Sum

At each position, decide: extend the current subarray, or start fresh? If the running sum is negative, it can only hurt — so reset.

current_sum = max_sum = nums[0] for num in nums[1:]: current_sum = max(num, current_sum + num) # extend or reset max_sum = max(max_sum, current_sum)
🧠 The DP Connection

Kadane's is actually DP in disguise! dp[i] = max(nums[i], dp[i-1] + nums[i]). Since dp[i] only depends on dp[i-1], we optimize the array to a single variable. Show the DP array first in interviews, then optimize — it demonstrates mastery.

⚠️ Common Bug

Initializing max_sum = 0 returns 0 for [-3, -2, -1] instead of the correct -1. Always initialize with nums[0].

🔤 String Manipulation Patterns

Strings are arrays of characters — but with their own quirks and patterns. Here's your cheat sheet:

📋 Pattern → Technique

  • 🔠 Anagram detection → Sort both or frequency count
  • 🪞 Palindrome check → Two pointers from edges inward
  • 📏 Longest common prefix → Vertical scan or sort
  • 🔢 String parsing (atoi) → State machine + overflow

🐍 Python String Tips

  • ⚠️ Strings are immutable — += in a loop is O(n²)
  • ✅ Use "".join(parts) instead — O(n) total
  • ✅ Counter is your best friend for freq counting
  • ✅ ord(c) - ord('a') → 0–25 index for arrays

Valid Anagram — Two Approaches

from collections import Counter def is_anagram(s: str, t: str) -> bool: return Counter(s) == Counter(t) # Or with a fixed-size array (faster for lowercase): def is_anagram_array(s: str, t: str) -> bool: if len(s) != len(t): return False freq = [0] * 26 for a, b in zip(s, t): freq[ord(a) - ord('a')] += 1 freq[ord(b) - ord('a')] -= 1 return all(f == 0 for f in freq)

Palindrome Check

def is_palindrome(s: str) -> bool: left, right = 0, len(s) - 1 while left < right: if s[left] != s[right]: return False left += 1 right -= 1 return True # O(n) time, O(1) space — no need to reverse!

Problem 1: Two Sum II — Sorted Array

Given a sorted array and a target, find two numbers that add up to target. Return 1-indexed positions.

def two_sum_sorted(numbers, target): left, right = 0, len(numbers) - 1 while left < right: current_sum = numbers[left] + numbers[right] if current_sum == target: return [left + 1, right + 1] elif current_sum < target: left += 1 else: right -= 1 return []

➕ Compute the sum of both elements
 
✅ Perfect match → return (1-indexed)

⬇️ Sum too big → move right left
 

Problem 2: Container With Most Water

Find two lines forming the container holding the most water. Move the shorter side inward.

def max_area(height): left, right = 0, len(height) - 1 best = 0 while left < right: width = right - left h = min(height[left], height[right]) best = max(best, h * width) if height[left] < height[right]: left += 1 else: right -= 1 return best

📏 Width = distance between pointers

Problem 3: Max Sum Subarray of Size K

Find the maximum sum of any contiguous subarray of exactly size k. Classic fixed sliding window.

def max_sum_subarray_k(nums, k): window_sum = sum(nums[:k]) max_sum = window_sum for i in range(k, len(nums)): window_sum += nums[i] - nums[i - k] max_sum = max(max_sum, window_sum) return max_sum

Problem 6: Maximum Subarray — Kadane's

At each element: extend the running subarray, or start fresh? If running sum is negative, reset.

def max_subarray(nums): current_sum = max_sum = nums[0] for num in nums[1:]: current_sum = max(num, current_sum + num) max_sum = max(max_sum, current_sum) return max_sum # [-2, 1, -3, 4, -1, 2, 1, -5, 4] # Answer: 6 → subarray [4, -1, 2, 1]

🤔 Extend current subarray OR start fresh?
