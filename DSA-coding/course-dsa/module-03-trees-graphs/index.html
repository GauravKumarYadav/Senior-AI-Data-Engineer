🌳 Tree Fundamentals & BST

Trees and graphs are where interviews separate "can code" from "can think recursively and model real systems." Let's start with the building blocks.

Binary Tree Terminology

8 ← root (depth 0) / \ 3 10 ← depth 1 / \ \ 1 6 14 ← depth 2 (leaves: 1, 6, 14) / \ / 4 7 13 ← depth 3

🌱 Root

The topmost node — no parent. Every tree has exactly one.

🍃 Leaf

A node with no children. The endpoints of the tree.

📏 Depth

Number of edges from root to that node. Root has depth 0.

📐 Height

Max depth of any leaf. A single-node tree has height 0.

⚖️ Balanced

Height of left and right subtrees differ by ≤ 1 at every node.

✅ Complete

Every level full except possibly the last, which fills left-to-right.

Binary Search Tree Property

For every node: left subtree values < node.value < right subtree values. This gives O(log n) search, insert, and delete — if balanced. Worst case (sorted input), it degenerates into a linked list: O(n).

The Node Class

class TreeNode: """Standard binary tree node used throughout this module.""" def __init__(self, val: int = 0, left: 'TreeNode | None' = None, right: 'TreeNode | None' = None) -> None: self.val = val self.left = left self.right = right
💡 Why BSTs Degenerate

Inserting [1, 2, 3, 4, 5] into a BST produces a linked list. This is why self-balancing trees exist: AVL trees, Red-Black trees (Java's TreeMap), and B-trees (databases). In Python, sortedcontainers.SortedList provides a balanced BST-like interface.

🌊 BFS — Level-Order Traversal

BFS explores nodes level by level using a queue (FIFO). It visits all nodes at depth d before any node at depth d + 1.

BFS Template

from collections import deque def bfs(root: TreeNode | None) -> list[list[int]]: if not root: return [] result: list[list[int]] = [] queue: deque[TreeNode] = deque([root]) while queue: level_size = len(queue) # nodes at current level level: list[int] = [] for _ in range(level_size): node = queue.popleft() level.append(node.val) if node.left: queue.append(node.left) if node.right: queue.append(node.right) result.append(level) return result

When to Use BFS

📊 Level-Order Traversal

Process nodes level by level — zigzag, right side view, level averages.

📏 Shortest Path (Unweighted)

BFS finds it naturally since it explores in order of distance.

🌱 Minimum Depth

BFS hits the first leaf at the shallowest depth — stop early!

💾 Serialization

Level-order is often the most natural tree serialization format.

BFS for Graphs (with visited set)

def bfs_graph(start: str, graph: dict[str, list[str]]) -> list[str]: visited: set[str] = {start} queue: deque[str] = deque([start]) order: list[str] = [] while queue: node = queue.popleft() order.append(node) for neighbor in graph[node]: if neighbor not in visited: visited.add(neighbor) queue.append(neighbor) return order
🔥 Airflow Connection

BFS is how Airflow's scheduler explores DAG dependencies to determine which tasks are ready to execute. Tasks with all upstream dependencies completed (in-degree = 0) get queued first — exactly like BFS processing nodes level by level. If you're interviewing for DE roles, drop this connection!

🔍 DFS — Depth-First Search

DFS explores as deep as possible along each branch before backtracking. It uses a stack (explicit or implicit via recursion).

Three Traversal Orders

1 Pre-order: 1, 2, 4, 5, 3 / \ In-order: 4, 2, 5, 1, 3 2 3 Post-order: 4, 5, 2, 3, 1 / \ 4 5

📋 Order & Mnemonic

  • 🔴 Pre-order: Node → Left → Right
  • 🟢 In-order: Left → Node → Right
  • 🔵 Post-order: Left → Right → Node

🎯 Use Cases

  • 🔴 Copying/serializing a tree
  • 🟢 BST gives sorted order
  • 🔵 Deleting a tree, evaluating expressions

Recursive Implementations

def preorder(root: TreeNode | None) -> list[int]: if not root: return [] return [root.val] + preorder(root.left) + preorder(root.right) def inorder(root: TreeNode | None) -> list[int]: if not root: return [] return inorder(root.left) + [root.val] + inorder(root.right) def postorder(root: TreeNode | None) -> list[int]: if not root: return [] return postorder(root.left) + postorder(root.right) + [root.val]

Iterative DFS (Pre-order with Explicit Stack)

def preorder_iterative(root: TreeNode | None) -> list[int]: if not root: return [] result: list[int] = [] stack: list[TreeNode] = [root] while stack: node = stack.pop() result.append(node.val) # Push right first so left is processed first (LIFO) if node.right: stack.append(node.right) if node.left: stack.append(node.left) return result
🎯 When to Use DFS

Path problems ("find all paths," "path sum = K"), tree validation (BST validation via in-order), backtracking (DFS is the backbone), and connected components (DFS from each unvisited node). If the problem has a recursive structure, DFS is your friend.

📊 Topological Sort & Graphs

A topological ordering of a DAG is a linear ordering of vertices such that for every edge u → v, vertex u comes before v. It answers: "In what order should I process tasks with dependencies?"

Kahn's Algorithm (BFS-Based)

from collections import deque def topological_sort_kahn(num_nodes: int, edges: list[tuple[int, int]]) -> list[int]: """Kahn's algorithm — BFS-based topological sort. Returns topological order, or empty list if cycle detected. Time: O(V + E) | Space: O(V + E)""" graph: dict[int, list[int]] = {i: [] for i in range(num_nodes)} in_degree = [0] * num_nodes for prereq, dependent in edges: graph[prereq].append(dependent) in_degree[dependent] += 1 # Start with nodes that have no prerequisites queue: deque[int] = deque( node for node in range(num_nodes) if in_degree[node] == 0 ) order: list[int] = [] while queue: node = queue.popleft() order.append(node) for neighbor in graph[node]: in_degree[neighbor] -= 1 if in_degree[neighbor] == 0: queue.append(neighbor) return order if len(order) == num_nodes else [] # cycle!

Graph Representations

📋 Adjacency List

  • 💾 Space: O(V + E)
  • 🔍 Edge check: O(degree)
  • ✅ Best for sparse graphs
  • ✅ Fast traversal (BFS/DFS)
  • 🐍 dict[str, list[str]]

📊 Adjacency Matrix

  • 💾 Space: O(V²)
  • 🔍 Edge check: O(1)
  • ✅ Best for dense graphs
  • ✅ Floyd-Warshall, small graphs
  • 🐍 list[list[int]]

Grid as Implicit Graph

DIRECTIONS = [(0, 1), (0, -1), (1, 0), (-1, 0)] # R, L, D, U def get_neighbors(row, col, rows, cols): neighbors = [] for dr, dc in DIRECTIONS: nr, nc = row + dr, col + dc if 0 <= nr < rows and 0 <= nc < cols: neighbors.append((nr, nc)) return neighbors
🔥 Airflow DAGs — This Is Literally Topological Sort

Each Airflow DAG is a directed acyclic graph. The scheduler performs a topological sort to determine task execution order. Tasks with no unfinished dependencies run first — this is exactly Kahn's algorithm.

In an interview, say: "Topological sort is how Airflow determines task execution order. Each task's dependencies must complete before it runs, and Kahn's algorithm naturally handles this by processing tasks whose in-degree reaches zero."

🔗 Union-Find & Dijkstra's

Union-Find manages disjoint sets with near-O(1) operations. Dijkstra's finds shortest paths in weighted graphs. Both are essential graph tools.

Union-Find (Disjoint Set)

class UnionFind: """Path compression + union by rank. O(α(n)) ≈ O(1) per op.""" def __init__(self, n: int) -> None: self.parent = list(range(n)) self.rank = [0] * n self.components = n def find(self, x: int) -> int: if self.parent[x] != x: self.parent[x] = self.find(self.parent[x]) # path compression return self.parent[x] def union(self, x: int, y: int) -> bool: root_x, root_y = self.find(x), self.find(y) if root_x == root_y: return False if self.rank[root_x] < self.rank[root_y]: root_x, root_y = root_y, root_x self.parent[root_y] = root_x if self.rank[root_x] == self.rank[root_y]: self.rank[root_x] += 1 self.components -= 1 return True

Union-Find Use Cases

🔗 Connected Components

Group nodes in undirected graphs. Track count with self.components.

🔄 Cycle Detection

If union(u, v) returns False, the edge creates a cycle.

🌲 Kruskal's MST

Process edges by weight, union vertices. Skip edges that create cycles.

⚡ Dynamic Connectivity

Efficiently answer "are A and B connected?" as edges are added.

Dijkstra's Algorithm

import heapq def dijkstra(graph: dict[str, list[tuple[str, int]]], start: str) -> dict[str, int]: """Shortest path from start to all reachable nodes. Time: O((V + E) log V) | Space: O(V)""" distances: dict[str, int] = {start: 0} heap: list[tuple[int, str]] = [(0, start)] while heap: dist, node = heapq.heappop(heap) if dist > distances.get(node, float("inf")): continue # already found shorter path for neighbor, weight in graph[node]: new_dist = dist + weight if new_dist < distances.get(neighbor, float("inf")): distances[neighbor] = new_dist heapq.heappush(heap, (new_dist, neighbor)) return distances
💡 Dijkstra's Key Limitation

Doesn't work with negative edge weights — use Bellman-Ford instead. You rarely need to implement Dijkstra's from scratch in a 45-minute interview. But knowing when to apply it (priority queue + relaxation) is essential.

Problem 2: Maximum Depth of Binary Tree

Return the maximum depth. Recursive DFS — elegant and simple.

def max_depth(root): if not root: return 0 left_depth = max_depth(root.left) right_depth = max_depth(root.right) return 1 + max(left_depth, right_depth) # O(n) time, O(h) space (recursion stack) # h = O(log n) balanced, O(n) skewed
📌 Define: depth of tree rooted here
🛑 Empty tree has depth 0
 
 
⬅️ How deep is the left side?
➡️ How deep is the right side?
 
🏆 1 (this node) + deeper subtree
 
⚡ Solve this in 2 mins — build confidence
⚠️ Python recursion limit = 1000

Problem 3: Validate Binary Search Tree

Pass a valid range (low, high) down the tree. Each node must satisfy low < val < high.

def is_valid_bst(root): def validate(node, low, high): if not node: return True if not (low < node.val < high): return False return ( validate(node.left, low, node.val) and validate(node.right, node.val, high) ) return validate(root, float("-inf"), float("inf")) # O(n) time, O(h) space
📌 Helper with valid range bounds
 
✅ Empty subtree is always valid
❌ Value outside valid range?
🛑 BST violation — reject!
 
⬅️ Left: tighten upper bound to node.val
➡️ Right: tighten lower bound to node.val
 
🚀 Start with (-∞, +∞)
 
⚠️ Don't just check immediate children!

🧩 Solutions: Problems 4–6

Problem 4: Number of Islands

Count connected components of '1's in a 2D grid. DFS flood fill — "sink" each island as you find it.

def num_islands(grid): if not grid: return 0 rows, cols = len(grid), len(grid[0]) count = 0 def dfs(r, c): if (r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] != '1'): return grid[r][c] = '0' # sink it dfs(r+1, c); dfs(r-1, c) dfs(r, c+1); dfs(r, c-1) for r in range(rows): for c in range(cols): if grid[r][c] == '1': count += 1 dfs(r, c) return count # O(m·n) time & space
📌 Edge case: empty grid
 
 
 
🔍 DFS: sink connected land
🛑 Out of bounds or water? Stop
 
 
💧 Mark as visited (sink the land)
↕️↔️ Explore all 4 directions
 
🔄 Scan every cell
 
🏝️ Found land! New island
🌊 Sink the entire island via DFS
 
⚠️ Modifies input — mention side effects!

Problem 5: Course Schedule (Cycle Detection)

Can you finish all courses? Build dependency graph → Kahn's algorithm → check if all nodes processed.

def can_finish(num_courses, prerequisites): graph = [[] for _ in range(num_courses)] in_degree = [0] * num_courses for course, prereq in prerequisites: graph[prereq].append(course) in_degree[course] += 1 queue = deque( i for i in range(num_courses) if in_degree[i] == 0) completed = 0 while queue: course = queue.popleft() completed += 1 for next_c in graph[course]: in_degree[next_c] -= 1 if in_degree[next_c] == 0: queue.append(next_c) return completed == num_courses
📌 Build adjacency list + in-degree
 
 
🔗 prereq → course (directed edge)
 
 
🚀 Start with no-prereq courses
 
 
 
 
🔄 Process ready courses (Kahn's)
✅ One more course completed
➡️ Reduce dependents' in-degree
 
🆓 Dependent now free? Enqueue it
 
🎯 All done? No cycle! (= Airflow DAG is valid)

Problem 6: Lowest Common Ancestor (BST)

Exploit BST ordering: if both values are smaller, go left. Both larger, go right. Otherwise — you're at the split point.

def lowest_common_ancestor(root, p, q): node = root while node: if p.val < node.val and q.val < node.val: node = node.left elif p.val > node.val and q.val > node.val: node = node.right else: return node return None # O(h) time, O(1) space — beautifully simple # For general binary tree: O(n) recursive DFS
📌 Start at root, walk down
 
 
⬅️ Both smaller → LCA is in left subtree
 
➡️ Both larger → LCA is in right subtree
 
🎯 Split point! One left, one right
✅ This node IS the LCA
 
⚡ Always ask: "BST or general tree?"
💡 BST = O(h), general = O(n)

🗺️ Tree vs Graph Recognition

🌳 "Binary tree," "root"

DFS (recursive) or BFS. First instinct: recursive DFS.

🔍 "BST," "sorted," "validate"

In-order traversal or range checking. O(h) solutions exist.

📊 "Level order," "min depth"

BFS with level tracking. level_size = len(queue) is key.

🏝️ "Grid," "islands," "connected"

DFS/BFS flood fill on implicit graph. 4-directional neighbors.

📋 "Prerequisites," "dependencies"

Topological sort (Kahn's). Think Airflow DAGs!

🔗 "Connected components"

Union-Find or DFS. Union-Find is better for dynamic edges.

📏 "Shortest path" (weighted)

Dijkstra's (non-negative weights). Priority queue + relaxation.

🔄 "Cycle detection"

Directed → topological sort / DFS states. Undirected → Union-Find.
