🔍 Binary Search — The Template

"Give me six hours to chop down a tree and I will spend the first four sharpening the axe." — Abraham Lincoln (on choosing the right algorithm)

Binary search is deceptively simple to describe and notoriously tricky to implement without off-by-one errors. Given a sorted array, eliminate half the search space each step — O(log n) time. Searching 1 billion elements takes only ~30 comparisons.

The Template (Inclusive Bounds)

def binary_search(nums: list[int], target: int) -> int: """Return index of target, or -1 if not found.""" left, right = 0, len(nums) - 1 while left <= right: mid = left + (right - left) // 2 # Avoids overflow in other languages if nums[mid] == target: return mid elif nums[mid] < target: left = mid + 1 else: right = mid - 1 return -1

Python's bisect Module

from bisect import bisect_left, bisect_right nums = [1, 3, 3, 3, 5, 7] bisect_left(nums, 3) # → 1 (first index where value >= 3) bisect_right(nums, 3) # → 4 (first index where value > 3) # bisect_left → first occurrence # bisect_right → position after last occurrence

Search Space Reduction Pattern

Binary search isn't limited to sorted arrays. It works on any monotonic predicate: a function that returns False, False, ..., True, True. You're searching for the boundary.

def binary_search_on_answer(lo: int, hi: int) -> int: """Find the smallest value where condition(mid) is True.""" while lo < hi: mid = lo + (hi - lo) // 2 if condition(mid): hi = mid # mid might be the answer else: lo = mid + 1 # mid is too small return lo
💡 The Monotonic Predicate Insight

Whenever you see "minimum capacity to ship in D days," "sqrt(x)," or "Koko eating bananas" — you're binary searching on the answer space, not an array. The predicate is monotonic: if speed=5 works, speed=6 also works. Binary search finds the boundary.

⚔️ Merge Sort vs Quick Sort

The two divide-and-conquer sorting titans. Know both cold — interviewers love asking why you'd choose one over the other.

🔀 Merge Sort

  • 📊 Always O(n log n) — no worst-case degradation
  • ✅ Stable sort (preserves relative order)
  • ⚠️ O(n) extra space for merging
  • 🏗️ Foundation for external sorting (data > RAM)
  • 🔗 Merge step reused in "Merge K Sorted Lists"
  • 📦 Divide → recurse → merge two sorted halves

⚡ Quick Sort

  • 📊 O(n log n) average, O(n²) worst case
  • ❌ Not stable
  • ✅ In-place — O(log n) stack space
  • 🚀 Faster in practice due to cache locality
  • 🔀 Partition step reused in Quick Select
  • 📦 Choose pivot → partition → recurse each side

Merge Sort Code

def merge_sort(arr: list[int]) -> list[int]: if len(arr) <= 1: return arr mid = len(arr) // 2 left = merge_sort(arr[:mid]) right = merge_sort(arr[mid:]) return _merge(left, right) def _merge(left, right): result, i, j = [], 0, 0 while i < len(left) and j < len(right): if left[i] <= right[j]: # <= ensures stability result.append(left[i]); i += 1 else: result.append(right[j]); j += 1 result.extend(left[i:]) result.extend(right[j:]) return result

Lomuto Partition (Quick Sort)

def _lomuto_partition(arr, lo, hi): """Partition using last element as pivot. Return pivot index.""" pivot = arr[hi] i = lo - 1 for j in range(lo, hi): if arr[j] <= pivot: i += 1 arr[i], arr[j] = arr[j], arr[i] arr[i + 1], arr[hi] = arr[hi], arr[i + 1] return i + 1 import random def quick_sort(arr, lo=0, hi=None): if hi is None: hi = len(arr) - 1 if lo < hi: rand_idx = random.randint(lo, hi) # randomize to avoid worst case arr[rand_idx], arr[hi] = arr[hi], arr[rand_idx] p = _lomuto_partition(arr, lo, hi) quick_sort(arr, lo, p - 1) quick_sort(arr, p + 1, hi)
💡 Why Not Always Use Merge Sort?

The O(n) extra space is a real cost. Quick Sort is in-place and faster in practice due to cache locality — accessing contiguous memory beats pointer-chasing. Python's built-in sorted() uses Tim Sort, a merge sort + insertion sort hybrid that's O(n) on nearly-sorted data.

🏔️ Heaps & the Top-K Pattern

A heap is a complete binary tree stored as an array where every parent satisfies the heap property. Python's heapq provides a min-heap. For max-heap, negate values.

import heapq nums = [5, 3, 8, 1, 2] heapq.heapify(nums) # O(n) — in-place heapq.heappush(nums, 4) # O(log n) smallest = heapq.heappop(nums) # O(log n) → returns 1 heapq.heapreplace(nums, 6) # Pop smallest, push 6 (one op) heapq.nsmallest(3, nums) # O(n log 3) — efficient for small k # Max-heap trick: negate on push, negate on pop max_heap = [] heapq.heappush(max_heap, -val) largest = -heapq.heappop(max_heap)

The Top-K Pattern

Top-K Largest → Min-heap of size K. The heap's root is the smallest of the K largest. Anything bigger than the root evicts it. Time: O(n log k).

def top_k_largest(nums: list[int], k: int) -> list[int]: heap = nums[:k] heapq.heapify(heap) for num in nums[k:]: if num > heap[0]: heapq.heapreplace(heap, num) return heap # unordered; sort if order matters

When to Use a Heap

🔄 Dynamic Min/Max

Repeated access to extreme values in a changing dataset → heap. Static? Just sort once.

🏆 Top-K Elements

K elements from a stream or large collection. Min-heap of size K → O(n log k).

🔗 Merging K Sorted

Min-heap of K heads. Always pick the smallest → O(N log k) total.

📋 Priority Scheduling

Task queues, Dijkstra's algorithm, meeting rooms — anything needing priority order.

🧠 Why Min-Heap for Largest?

Counter-intuitive but elegant: the min-heap root is a gatekeeper. It's the smallest thing allowed in the "top K club." Anything bigger kicks it out. At the end, everyone in the club is top-K. This is the key insight interviewers test.

🧩 Binary Search Classic + Rotated

Problem 1: Binary Search (LeetCode 704)

Sorted array + target → return index or -1. The warmup — but mention left + (right - left) // 2 to avoid overflow in Java/C++.

def search(nums, target): left, right = 0, len(nums) - 1 while left <= right: mid = left + (right - left) // 2 if nums[mid] == target: return mid elif nums[mid] < target: left = mid + 1 else: right = mid - 1 return -1 # O(log n) time, O(1) space
📌 Classic binary search on sorted array
 
📏 Inclusive bounds: left and right both valid
 
🔄 While search space is non-empty:
📊 Calculate midpoint (overflow-safe)
 
✅ Found it → return index
 
➡️ Target is bigger → search right half
 
⬅️ Target is smaller → search left half
 
❌ Exhausted → not found

Problem 2: Search in Rotated Sorted Array (LeetCode 33)

Array rotated at unknown pivot: [4,5,6,7,0,1,2]. At any mid point, one half is always sorted. Determine which, then check if target lies within it.

def search_rotated(nums, target): left, right = 0, len(nums) - 1 while left <= right: mid = left + (right - left) // 2 if nums[mid] == target: return mid # Left half is sorted if nums[left] <= nums[mid]: if nums[left] <= target < nums[mid]: right = mid - 1 else: left = mid + 1 # Right half is sorted else: if nums[mid] < target <= nums[right]: left = mid + 1 else: right = mid - 1 return -1 # O(log n) time, O(1) space
📌 Binary search in rotated sorted array
 
 
🔄 Standard binary search loop
 
✅ Found → return
 
🔑 KEY: nums[left] <= nums[mid]?
⬅️ Left half is sorted
🎯 Target in sorted left? → go left
➡️ Otherwise → go right
 
➡️ Right half is sorted
🎯 Target in sorted right? → go right
⬅️ Otherwise → go left
 
❌ Not found
💡 With duplicates (LC 81): worst case O(n)
🎯 The Critical Insight

nums[left] <= nums[mid] determines which half is sorted. Walk through [4,5,6,7,0,1,2] with target 0 on the whiteboard. If duplicates are allowed, you may need left += 1 to skip them — worst case degrades to O(n).

🧩 Kth Largest & Top K Frequent

Problem 3: Kth Largest Element (LeetCode 215)

Two approaches — present both and ask the interviewer which trade-off they prefer.

# Approach 1: Min-Heap of Size K import heapq def findKthLargest(nums, k): heap = nums[:k] heapq.heapify(heap) for num in nums[k:]: if num > heap[0]: heapq.heapreplace(heap, num) return heap[0] # O(n log k) time, O(k) space # Approach 2: Quick Select def findKthLargest_qs(nums, k): target = len(nums) - k lo, hi = 0, len(nums) - 1 while lo < hi: p = partition(lo, hi) if p == target: return nums[p] elif p < target: lo = p + 1 else: hi = p - 1 return nums[lo] # O(n) avg / O(n²) worst, O(1) space
🏔️ Heap approach:
 
📦 Seed heap with first K elements
🔄 For each remaining element:
🚪 Bigger than gatekeeper? Swap in
🏆 Root = Kth largest
⏱️ Consistent O(n log k)
 
⚡ Quick Select approach:
🔄 Convert "Kth largest" → index n-k
📊 Partition array around random pivot
✅ Pivot lands at target? Done!
➡️ Pivot too small? Search right
⬅️ Pivot too big? Search left
⚡ O(n) average but O(n²) worst
 
💡 Pragmatic: heapq.nlargest(k, nums)[-1]

Problem 4: Top K Frequent Elements (LeetCode 347)

from collections import Counter import heapq def topKFrequent(nums, k): """Min-heap approach: O(n log k)""" counts = Counter(nums) heap = [] for num, freq in counts.items(): if len(heap) < k: heapq.heappush(heap, (freq, num)) elif freq > heap[0][0]: heapq.heapreplace(heap, (freq, num)) return [num for _, num in heap] def topKFrequent_bucket(nums, k): """Bucket sort approach: O(n)""" counts = Counter(nums) buckets = [[] for _ in range(len(nums)+1)] for num, freq in counts.items(): buckets[freq].append(num) result = [] for freq in range(len(buckets)-1, 0, -1): for num in buckets[freq]: result.append(num) if len(result) == k: return result return result
🏔️ Heap approach:
 
📊 Count frequencies first
📦 Min-heap keyed by frequency
🔄 Heap < K? Push freely
🚪 Higher freq than gatekeeper? Swap in
🏆 Heap has top K frequent elements
 
🪣 Bucket sort approach:
📊 Count frequencies
📦 bucket[freq] = list of nums with that freq
🔄 Max freq = len(nums), so finite buckets
⬇️ Walk buckets from high freq down
🏆 Collect until we have K elements
 
💡 Bucket sort is O(n) — great follow-up!
💡 The Bucket Sort Flex

The bucket sort approach is O(n) and very elegant. Explain: "The maximum possible frequency is n, so I can use frequency as an array index." This is a great optimization follow-up that impresses interviewers every time.

🧩 Merge K Lists & Find Median

Problem 5: Merge K Sorted Lists (LeetCode 23)

Use a min-heap to always pick the smallest current head among all K lists. Push the next node back into the heap.

import heapq from dataclasses import dataclass @dataclass class ListNode: val: int = 0 next: 'ListNode | None' = None def __lt__(self, other): return self.val < other.val # Required for heap! def mergeKLists(lists): heap = [head for head in lists if head] heapq.heapify(heap) dummy = current = ListNode() while heap: smallest = heapq.heappop(heap) current.next = smallest current = current.next if smallest.next: heapq.heappush(heap, smallest.next) return dummy.next # O(N log k) time — N = total nodes, k = number of lists # O(k) space — heap holds at most k nodes

Problem 6: Find Median from Data Stream (LeetCode 295)

🧠 Two-Heap Diagram

Visualize two heaps pointing at each other: ▽ max_heap (small half) | min_heap (large half) △. The roots face each other and the median lives at the boundary. Max-heap root = largest of small half. Min-heap root = smallest of large half. Median = one root or average of both.

import heapq class MedianFinder: def __init__(self): self.lo = [] # Max-heap (negated) — small half self.hi = [] # Min-heap — large half def addNum(self, num): heapq.heappush(self.lo, -num) # Ensure max of lo <= min of hi if self.hi and -self.lo[0] > self.hi[0]: heapq.heappush(self.hi, -heapq.heappop(self.lo)) # Balance sizes: lo can have at most 1 more than hi if len(self.lo) > len(self.hi) + 1: heapq.heappush(self.hi, -heapq.heappop(self.lo)) elif len(self.hi) > len(self.lo): heapq.heappush(self.lo, -heapq.heappop(self.hi)) def findMedian(self): if len(self.lo) > len(self.hi): return -self.lo[0] return (-self.lo[0] + self.hi[0]) / 2.0 # addNum: O(log n) | findMedian: O(1) | Space: O(n)

Walkthrough: Adding [2, 3, 1, 5, 4]

Step │ Add │ lo (max-heap) │ hi (min-heap) │ Median ─────┼─────┼───────────────┼───────────────┼─────── 1 │ 2 │ [2] │ [] │ 2.0 2 │ 3 │ [2] │ [3] │ 2.5 3 │ 1 │ [2, 1] │ [3] │ 2.0 4 │ 5 │ [2, 1] │ [3, 5] │ 2.5 5 │ 4 │ [3, 2, 1] │ [4, 5] │ 3.0

🎯 The Three Pillars

Binary Search reduces O(n) to O(log n) — always ask "Is the search space monotonic?" Divide and Conquer (Merge/Quick Sort) breaks problems into independent subproblems — the foundation of O(n log n). Heaps give O(log n) access to extremes — whenever you hear "top K," "median," or "merge sorted," reach for heapq.
