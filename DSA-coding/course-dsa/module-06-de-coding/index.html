Pandas vs PySpark: When to Use Which

🐼 Pandas

  • 💻 Single-machine, in-memory
  • 📊 Ideal for data ≤ ~10 GB
  • ⚡ Faster iteration, richer API
  • 🐛 Easier debugging (eager eval)
  • 🔬 Prototyping & local exploration
  • 🧪 Great for notebooks & ad-hoc

⚡ PySpark

  • 🌐 Distributed across a cluster
  • 📊 Necessary for terabytes+
  • 😴 Lazy evaluation → optimized DAG
  • 🏭 Production ETL pipelines
  • ☁️ Databricks / EMR / Dataproc
  • 📋 Always provide explicit schema!

📜 Parse Logs & Deduplicate Events

1. Parse & Aggregate Log Files

10 GB web-server log → extract level & service → count (level, service) pairs → top 5 errors.

🐼 Pandas
import pandas as pd, re pattern = (r"(?P<ts>\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})" r"\s+(?P<level>\w+)\s+\[(?P<service>\w+)\]" r"\s+(?P<msg>.*)") with open("server.log") as f: records = [m.groupdict() for line in f if (m := re.match(pattern, line))] df = pd.DataFrame(records) df["ts"] = pd.to_datetime(df["ts"]) top_errors = ( df[df["level"] == "ERROR"] .groupby(["level", "service"]) .size().reset_index(name="count") .sort_values("count", ascending=False) .head(5) )
⚡ PySpark
from pyspark.sql.functions import ( regexp_extract, col, desc) raw = spark.read.text("server.log") pat = (r"(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})" r"\s+(\w+)\s+\[(\w+)\]\s+(.*)") parsed = raw.select( regexp_extract("value",pat,1).alias("ts"), regexp_extract("value",pat,2).alias("level"), regexp_extract("value",pat,3).alias("service"), ).filter(col("level") != "") top_errors = ( parsed.filter(col("level") == "ERROR") .groupBy("level", "service") .count() .orderBy(desc("count")) .limit(5) )

2. Deduplicate Events

Event stream with at-least-once delivery → keep only the latest event per key.

🐼 Pandas
# Sort + drop_duplicates deduped = ( df.sort_values("event_time", ascending=False) .drop_duplicates(subset=["key"], keep="first") .sort_values("key") ) # Alternative: explicit idxmax idx = df.groupby("key")["event_time"].idxmax() deduped = df.loc[idx]
⚡ PySpark
from pyspark.sql.window import Window from pyspark.sql.functions import ( row_number, col, desc) w = (Window.partitionBy("key") .orderBy(desc("event_time"))) deduped = ( df.withColumn("rn", row_number().over(w)) .filter(col("rn") == 1) .drop("rn") )
💡 Interview Tip

Always clarify tie-breaking: "If two events share the same timestamp, which wins?" In PySpark, dropDuplicates() is non-deterministic — it doesn't let you control which row is kept. Always prefer the ROW_NUMBER() window function approach when ordering matters.

🎯 Sessionization — The #1 DE Question

Given user click events, group them into sessions. A new session starts when the gap between consecutive events exceeds 30 minutes.

🧠 THE Pattern: lag → gap → flag → cumsum

This is the #1 most-asked data engineering coding question. Practice it until you can write it from memory. The four-step pattern works identically in Pandas and PySpark: 1) Get previous event time with lag/shift → 2) Compute the gap → 3) Flag new sessions (gap > threshold or first event) → 4) Cumulative sum of flags = session ID.

🐼 Pandas
df = df.sort_values( ["user_id", "event_time"]) # 1. lag df["prev_time"] = ( df.groupby("user_id")["event_time"] .shift(1)) # 2. gap df["gap_min"] = ( (df["event_time"] - df["prev_time"]) .dt.total_seconds() / 60) # 3. flag df["new_session"] = ( df["gap_min"].isna() | (df["gap_min"] > 30)) # 4. cumsum → session_id df["session_id"] = ( df.groupby("user_id")["new_session"] .cumsum().astype(int))
⚡ PySpark
from pyspark.sql.functions import ( lag, col, when, unix_timestamp, sum as spark_sum) from pyspark.sql.window import Window w = (Window.partitionBy("user_id") .orderBy("event_time")) sessionized = ( df # 1. lag .withColumn("prev_time", lag("event_time").over(w)) # 2. gap .withColumn("gap_min", (unix_timestamp("event_time") - unix_timestamp("prev_time")) / 60) # 3. flag .withColumn("new_session", when(col("gap_min").isNull() | (col("gap_min") > 30), 1).otherwise(0)) # 4. cumsum → session_id .withColumn("session_id", spark_sum("new_session").over(w)) )

Common Variations

⏱️ Variable Timeout

Threshold depends on user's historical behavior (e.g., median gap × 2). Compute in a separate CTE and join.

📊 Session Metrics

After sessionizing: session duration, event count, bounce rate (sessions with only 1 event). Simple groupby on session_id.

📱 Cross-Device

User on multiple devices? Sessionize per (user_id, device_id) pair instead.

🏗️ Mini ETL & Data Reconciliation

Mini ETL Pipeline

Read CSV → clean (drop nulls, cast types, normalize strings) → transform (add revenue column) → write to Parquet.

🐼 Pandas
# ---- EXTRACT ---- df = pd.read_csv("sales.csv", dtype={"order_id": str, "product": str}, parse_dates=["order_date"]) # ---- CLEAN ---- critical = ["order_id","product","qty","price"] df = df.dropna(subset=critical) df["product"] = ( df["product"].str.strip().str.lower()) df["price"] = df["price"].round(2) # ---- TRANSFORM ---- df["revenue"] = df["qty"] * df["price"] # ---- LOAD ---- df.to_parquet("sales_clean.parquet", compression="snappy", index=False)
⚡ PySpark
from pyspark.sql.functions import ( col, trim, lower, round as spark_round) # ---- EXTRACT (explicit schema!) ---- df = (spark.read.option("header", True) .schema(schema) # enforce, don't infer .csv("sales.csv")) # ---- CLEAN ---- df = (df.dropna(subset= ["order_id","product","qty","price"]) .withColumn("product", lower(trim(col("product")))) .withColumn("price", spark_round(col("price"), 2))) # ---- TRANSFORM ---- df = df.withColumn("revenue", col("qty") * col("price")) # ---- LOAD ---- df.write.mode("overwrite").parquet( "sales_clean.parquet", compression="snappy")
💡 Always Provide Explicit Schema in PySpark

inferSchema=True triggers an extra pass over the data and can guess wrong types. Mentioning this unprompted wins you bonus points. Also: partition Parquet by date (partitionBy("order_date")) for efficient downstream queries.

Data Reconciliation (Anti-Joins)

Migrated Oracle → BigQuery. Find: rows missing from target, rows missing from source, value mismatches.

🐼 Pandas
key = "order_id" # Missing from target (anti-join) missing = source_df[ ~source_df[key].isin(target_df[key])] # Alt: merge with indicator merged = source_df.merge( target_df, on=key, how="left", indicator=True) missing = merged[ merged["_merge"] == "left_only"] # Value mismatches inner = source_df.merge( target_df, on=key, suffixes=("_src", "_tgt")) for c in value_cols: inner[f"{c}_match"] = ( inner[f"{c}_src"] == inner[f"{c}_tgt"])
⚡ PySpark
key = "order_id" # Anti-join: missing from target missing_in_target = source_df.join( target_df, on=key, how="left_anti") # Anti-join: missing from source missing_in_source = target_df.join( source_df, on=key, how="left_anti") # Value mismatches joined = (source_df.alias("s") .join(target_df.alias("t"), on=key, how="inner")) mismatch = None for c in value_cols: cond = col(f"s.{c}") != col(f"t.{c}") mismatch = cond if not mismatch \ else (mismatch | cond)
🔑 Use the Term "Anti-Join"

It tells the interviewer you know relational algebra. In PySpark, left_anti is purpose-built for this. In Pandas, simulate it with isin() negation or merge(how='left', indicator=True). Watch out: NULL != NULL evaluates to NULL, not TRUE — use eqNullSafe() in PySpark.

🌊 Stream Processing & GroupBy Aggs

Stream Processing (Generator Pattern)

Simulate a stream of events. Maintain running count and running average per category. The generator demonstrates lazy evaluation with constant memory.

from collections import defaultdict from typing import Generator, NamedTuple class Event(NamedTuple): category: str amount: float def event_stream() -> Generator[Event, None, None]: """Could read from Kafka, a socket, or a file of arbitrary size.""" events = [Event("electronics", 299.99), Event("books", 12.50), Event("electronics", 149.00), Event("books", 25.00)] yield from events def process_stream(stream): counts: dict[str, int] = defaultdict(int) totals: dict[str, float] = defaultdict(float) for event in stream: counts[event.category] += 1 totals[event.category] += event.amount avg = totals[event.category] / counts[event.category] print(f"[{event.category}] count={counts[event.category]} avg={avg:.2f}") # O(1) per event, O(k) space where k = unique categories

GroupBy Aggregations — Named Aggs

🐼 Pandas (Named Aggs)
agg_df = ( df.groupby("product") .agg( total_rev=("revenue", "sum"), avg_order=("revenue", "mean"), num_orders=("revenue", "count"), max_order=("revenue", "max"), std_rev=("revenue", "std"), ) ) agg_df["cv"] = ( agg_df["std_rev"] / agg_df["avg_order"]) agg_df = agg_df.drop(columns=["std_rev"]) # Custom agg example: def iqr(s): return s.quantile(.75)-s.quantile(.25) df.groupby("product").agg( revenue_iqr=("revenue", iqr))
⚡ PySpark
from pyspark.sql.functions import ( sum as _sum, avg as _avg, count as _count, max as _max, stddev, col) agg_df = ( df.groupBy("product") .agg( _sum("revenue").alias("total_rev"), _avg("revenue").alias("avg_order"), _count("revenue").alias("num_orders"), _max("revenue").alias("max_order"), stddev("revenue").alias("std_rev"), ) .withColumn("cv", col("std_rev") / col("avg_order")) .drop("std_rev") ) # Custom aggs need pandas_udf # UDFs are 2-10× slower than builtins!
💡 Named Aggs > Dict Style

In Pandas, prefer named aggregations (col=("src", "func")) over the old agg({"col": "func"}) dict style — it avoids MultiIndex headaches and reads cleaner. PySpark UDFs involve JVM↔Python serialization — 2-10× slower than builtins. Use them only when no built-in alternative exists.

🔄 Pivot/Unpivot & Time-Series

Pivot (Long → Wide) & Unpivot (Wide → Long)

🐼 Pandas
# ---- Pivot ---- wide = df.pivot_table( index="product", columns="quarter", values="revenue", aggfunc="sum", fill_value=0) wide.columns = [f"Q{q}" for q in wide.columns] wide = wide.reset_index() # ---- Unpivot ---- long = wide.melt( id_vars=["product"], value_vars=["Q1","Q2","Q3","Q4"], var_name="quarter", value_name="revenue")
⚡ PySpark
# ---- Pivot ---- wide = ( df.groupBy("product") .pivot("quarter", values=["Q1","Q2","Q3","Q4"]) .agg(_sum("revenue")) .fillna(0)) # ---- Unpivot (stack) ---- unpivot_expr = """ stack(4, 'Q1', Q1, 'Q2', Q2, 'Q3', Q3, 'Q4', Q4 ) as (quarter, revenue) """ long = (wide .select("product", expr(unpivot_expr)) .filter(col("revenue") != 0))
💡 Always Pass Pivot Values Explicitly

In PySpark, without the values= list, Spark runs an extra aggregation job to discover distinct values — a performance killer on large datasets. Long format is better for storage and joins; wide is better for readability and ML features.

Time-Series Resampling

Minute-level sensor data → hourly averages → forward-fill gaps → 7-period rolling average.

🐼 Pandas
df = df.set_index("timestamp") # Resample to hourly hourly = df["value"].resample("1h").mean() # Forward-fill gaps hourly = hourly.ffill() # Rolling 7-period average hourly_df = hourly.to_frame(name="avg") hourly_df["rolling_7"] = ( hourly_df["avg"].rolling(7).mean())
⚡ PySpark
from pyspark.sql.functions import ( window, avg as _avg, last) # Resample (tumbling window) hourly = (df .groupBy(window("timestamp", "1 hour")) .agg(_avg("value").alias("avg")) .withColumn("hour", col("window.start")) .drop("window").orderBy("hour")) # Forward-fill fill_w = Window.orderBy("hour").rowsBetween( Window.unboundedPreceding, Window.currentRow) hourly = hourly.withColumn("avg", last("avg", ignorenulls=True).over(fill_w)) # Rolling 7-period roll_w = Window.orderBy("hour").rowsBetween(-6,0) hourly = hourly.withColumn("rolling_7", _avg("avg").over(roll_w))

🔗 Join Patterns & Window Functions

Join Patterns

🐼 Pandas
# Left join result = orders.merge( customers, on="customer_id", how="left") result["name"] = result["name"].fillna("Unknown") # Self-join: orders within 24h sj = orders.merge( orders, on="customer_id", suffixes=("_a", "_b")) sj = sj[ (sj["order_id_a"] < sj["order_id_b"]) & (abs((sj["order_time_a"] - sj["order_time_b"]) .dt.total_seconds()) <= 86400)] # Broadcast: N/A in Pandas (all local) result = orders.merge( regions, on="region_id", how="left")
⚡ PySpark
from pyspark.sql.functions import broadcast # Left join result = (orders .join(customers, on="customer_id", how="left") .fillna({"name": "Unknown"})) # Self-join: orders within 24h o1, o2 = orders.alias("a"), orders.alias("b") sj = (o1.join(o2, (col("a.customer_id") == col("b.customer_id")) & (col("a.order_id") < col("b.order_id"))) .filter(spark_abs( unix_timestamp("a.order_time") - unix_timestamp("b.order_time")) <= 86400)) # Broadcast join (small dim table) result = orders.join( broadcast(regions), on="region_id", how="left")
💡 Always Mention broadcast() for Small Tables

In production, Spark's AQE may auto-broadcast, but explicitly marking it shows you understand the physical plan. Check with df.explain(True) — look for BroadcastHashJoin vs SortMergeJoin.

Window Functions

# PySpark — rank, lag, lead, running total from pyspark.sql.functions import rank, dense_rank, row_number, lag, lead, sum as _sum, desc dept_salary_w = Window.partitionBy("department").orderBy(desc("salary")) dept_date_w = Window.partitionBy("department").orderBy("hire_date") df = (df .withColumn("salary_rank", rank().over(dept_salary_w)) .withColumn("salary_dense_rank", dense_rank().over(dept_salary_w)) .withColumn("row_num", row_number().over(dept_salary_w)) .withColumn("prev_salary", lag("salary", 1).over(dept_date_w)) .withColumn("next_salary", lead("salary", 1).over(dept_date_w)) .withColumn("running_total", _sum("salary").over( dept_date_w.rowsBetween(Window.unboundedPreceding, Window.currentRow))) )
🧠 ROWS vs RANGE — The Subtle Difference

ROWS BETWEEN = physical rows. "The 3 rows before me" means exactly 3 rows. RANGE BETWEEN = logical values. With ties in ORDER BY salary, RANGE includes all tied rows at once — ROWS treats them separately. For running totals and moving averages, always use ROWS for deterministic results. This is one of the most common window-function bugs.

⚠️ Common Interview Mistakes

🐢 Using .apply() in Pandas

.apply() is a Python-level loop — 10-100× slower than vectorized alternatives. Always look for a built-in method first.

💾 Forgetting .cache()

If you reuse a DataFrame multiple times in PySpark (e.g., join it twice), call .cache() to avoid recomputing the entire DAG.

🕳️ Not Handling NULLs

In Pandas: NaN != NaN → True. In PySpark: NULL != NULL → NULL. Use .isna() / .isNull(), never ==.

💥 Collecting Large DFs

df.collect() pulls all data to the driver — instant OOM on large datasets. Use .show(), .take(n), or write to storage.

🔀 Ignoring Sort Order

drop_duplicates in Pandas respects the current sort. If you don't sort first, the "kept" row is arbitrary.

🔄 transform() vs agg()

transform returns same-length Series (broadcasts). agg returns reduced DataFrame. Wrong choice = wrong shape.
